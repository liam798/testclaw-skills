# 使用 TestClaw 通用模板

## 模板 1：配置 TestClaw CLI 并登录

```text
帮我配置 TestClaw CLI：
1. 使用 TestClaw Server 地址 `https://testclaw.dev.ad2.cc`
2. 执行 `testclaw config set base_url ...`
3. 指导我执行 `testclaw login`
4. 登录后用 `testclaw --json whoami` 验证当前用户
5. 如果失败，判断是服务地址错误、浏览器登录未完成还是登录态失效
```

## 模板 2：查看空闲设备并准备调试

```text
使用 TestClaw 帮我处理设备准备：
1. 先查看当前空闲或在线设备
2. 选择一台可用 Android 设备
3. 直接占用并准备 Android 调试
4. 告诉我设备信息、调试地址和后续可执行动作
5. 如果失败，明确说明是设备不足、登录态失效还是服务异常
```

## 模板 3：完整证据链 APK 自动化测试

```text
使用 TestClaw 自动化测试这个 APK：
1. 先找可复用 suite；没有 suite 时使用 testclaw-cli 手工冒烟模式
2. 全程录屏
3. 全程日志
4. 全程网络抓包
5. 关键节点截图
6. 性能采集
7. 生成结构化报告
8. 缺任何证据时标注阻塞或证据不完整
```

## 模板 4：执行套件并查看结果

```text
使用 TestClaw 帮我执行自动化任务：
1. 先复用已有项目、模块、用例、套件
2. 确认套件 evidence workflow 完整开启
3. 选择可用设备执行套件
4. 拉取执行结果、录屏、日志、抓包、截图和性能数据
5. 给我返回结构化结论，不要只说“执行了”
```

## 模板 5：UI 校对与真机巡检

```text
使用 TestClaw 帮我做 UI 校对：
1. 先找一台空闲设备并准备调试
2. 启动 evidence workflow
3. 打开目标应用或页面
4. 检查页面元素、文案、布局、错位、截断、遮挡
5. 输出问题清单、录屏、日志、抓包、截图、性能数据和简短结论
6. 结束后释放设备
```
